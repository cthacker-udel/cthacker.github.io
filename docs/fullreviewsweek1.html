  • Typescript
    • I started learning Typescript recently, following mostly the guide(very nice guide by the way!) posted by Dr. Bart under the Portfolio Description Page, specifically this book: Programming Typescript: O'Reilly. I am more of a visual learner and I like challenges along the way, I learn a lot by reading documents or tutorials online. This book has been fantastic from the first page, going over in depth about types in Typescript and how they differ from Javascript. I appreciate the attention to detail such as explaining how the TSC compiler works, and explaining the issues with JS and how TS fixes them. Such as instead of compiling their code at run-time, TS continuously compiles it's code to spot any type errors, and alert the programmer to them. I find this language to be labor-intensive at first(more typing, more bugs(not a bad thing!)), but as you get the hang of the language, the overall sense of freedom you feel when coding in this type of language. It is a lot more freedom then per-se Javascript. This topic about freedom appeared in an earlier class I took, CISC210(Introduction to Systems Programming), which touched on the topic of Freedom, and C is an absolute free language, considering there is no garbage collector, you have to define every type, you have to manage every piece of memory you borrow. The freedom lies in the programmer being able to feel in control with their computer, and knowing how to manipulate the BASH terminal embodies that feeling of being in control. The Typescript Language compared to Javascript has a stronger feeling of being in control, knowing when there is a bug in your code, knowing what type this variable is and declaring it as such, being able to control what types are passed into a function, what it returns. Those aspects of this language make the programmer feel more in control of what they are programming. Freedom is the absolute foundation of GNU, open-source for everyone and free. Back on topic, my learning style is practice, I reinforce all the material by practicing, whether its doing practice problems from the textbook, or finding websites online, or even creating my own side projects. I usually desire to have a strong foundation of the basics, a very firm understanding of the beginner-intermediate level code, and then begin to start reaching for advanced level style syntax. I found this website which has tons of practice problems, and even a island-themed progression, moving from island to island, leveling up, the website it called CheckiO.
